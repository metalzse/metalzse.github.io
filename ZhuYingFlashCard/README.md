# 注音練習卡

一個純前端的注音符號學習頁面。使用者直接用瀏覽器打開 `index.html` 就能開始練習，不需要安裝套件、啟動後端或連線到伺服器。

## 使用方式

1. 用瀏覽器開啟 `index.html`。
2. 頁面會隨機顯示一個注音符號。
3. 使用者自行判斷是否認得該注音。
4. 認得就按「對」，不認得就按「錯」。
5. 按下後會自動抽出下一個注音。
6. 按「結算」可進入本次練習結算頁。

也可以使用鍵盤操作：

- `Enter` 或 `ArrowRight`：答對
- `Backspace` 或 `ArrowLeft`：答錯

## 主要功能

- 隨機出題 37 個注音符號。
- 統計總練習次數、答對次數、答錯次數與有效速度樣本。
- 顯示每個注音的答對次數、答錯次數、平均答對時間與目前抽題權重。
- 提供結算頁，以注音字表方格顯示每個注音的答對率 heatmap 與總結分數。
- 將每個注音的練習資料保存在瀏覽器 `localStorage`。
- 可用右上角重設按鈕清除所有練習統計。
- 頁面固定在單一視窗高度內，資料清單在自己的區塊中捲動。

## 結算頁

主頁的「結算」按鈕會開啟 `summary.html`。此頁會顯示本次練習分數、總次數、答對、答錯、整體答對率，以及每個注音的方格統計。

分數採 0–100 分制。每個注音都各占一份，未練過是 0 分；有練過就看答對率距離 90% 目標有多近，90% 以上會拿滿該注音的分數。進入結算頁時，分數會先像角子老虎機一樣快速跳動，再停在真正分數。

每個方格包含：

- 注音符號
- 答對率
- 答對次數
- 答錯次數

方格底色代表答對率：

- 綠色：答對率達到 90% 以上
- 紅色：答對率較低，50% 會偏向紅色警示區
- 灰色：尚未回答

在結算頁按「回到練習」只會返回主頁，原本的 `localStorage` 練習統計會保留並繼續累積。

## 抽題加權規則

每個注音都有一個抽題權重，權重越高，越容易被抽到。

目前規則如下：

- 尚未回答過：`1.75x`
- 已回答且沒有答錯淨債：`0.72 ** max(1, 答對次數 - 答錯次數)`
- 答錯淨債大於 0：`2.15 ** (答錯次數 - 答對次數)`
- 最低權重：`0.16x`
- 最高權重：`30x`

範例：

| 狀態 | 權重 |
| --- | --- |
| 尚未回答 | `1.75x` |
| 對 1 / 錯 0 | `0.72x` |
| 對 2 / 錯 0 | `0.52x` |
| 對 1 / 錯 1 | `0.72x` |
| 對 0 / 錯 1 | `2.15x` |
| 對 0 / 錯 2 | `4.62x` |

這代表還沒練過的注音會優先被看見；已經答對的注音會逐步降低出現率；答錯越多的注音會以指數方式提高出現率。答對與答錯會互相抵銷。

## 答題時間

頁面會記錄使用者按下「對」之前花費的時間，並顯示每個注音的平均答對秒數。

目前答題時間只作為參考資料，不會影響抽題機率。超過 20 秒的答對時間會被視為可能離開頁面，不會納入平均時間樣本。

## 檔案結構

```text
.
├── index.html   # 頁面結構
├── summary.html # 本次練習結算頁
├── styles.css   # 版面與視覺樣式
├── app.js       # 注音資料、抽題邏輯、統計與 localStorage
├── summary.js   # 結算頁分數、heatmap 與返回主頁邏輯
└── README.md    # 專案說明
```

## 資料儲存

練習資料儲存在目前瀏覽器的 `localStorage`，key 為：

```text
zhuyin-flashcard-state-v1
```

資料只存在使用者本機瀏覽器中。換瀏覽器、清除網站資料或按下頁面上的重設按鈕，都會影響已保存的練習紀錄。
