# 兒童學習遊戲

這個專案是純前端的兒童學習遊戲入口頁，可直接部署到 GitHub Pages。

## 學習功能

- `TurnBasedRPG/`：純前端回合制 RPG 戰鬥原型；每場隨機產生 2～5 點敵人額度，一般敵人占 1 點、符文魔王占 2 點。符文魔王有 20% 機率登場，以玩家尚未學會的故事字建立獨立字庫；每次成功防守才新增一字。赤焰魔像限時 10 秒，苔岩巨像有 20% 機率雙擊，防禦題採用注音練習卡權重公式。
- `ZhuyinAdventure/`：注音冒險故事，透過閱讀故事、選擇路線與收集道具練習注音閱讀。
- `ZhuYingFlashCard/`：注音練習卡，隨機顯示注音符號，由使用者唸出發音並由旁人判斷答對或答錯。

## 主選單工具

- `匯入學會的字`：在最上層選單匯入先前輸出的 JSON 學會字清單（每筆包含 `character` 與 `zhuyin`），合併到共用學會字庫，供各學習功能使用；舊版純文字清單仍可匯入。
- `升級舊版字庫`：將舊版字串陣列轉換為新版物件格式，並從四個冒險故事補上可辨識的注音。無法推斷注音的外部字會保留為空白，不會刪除。
- `清除目前記錄`：在使用者確認後，清除共用學會字庫、注音冒險進度與背包、注音練習卡統計。

### 舊版字庫升級

1. 新版首頁載入時會檢查 `adventure-learned-characters-version`；沒有版本 2 標記即視為舊版資料。
2. 系統會自動建立故事字音對照表，將 `"人"` 轉換為 `{ "character": "人", "zhuyin": "ㄖㄣˊ" }`。
3. 轉換後立即寫回原本的瀏覽器儲存位置並標記版本 2，因此只會自動執行一次。
4. 完成時會顯示總字數、成功補上的注音數，以及仍無法判斷的字。
5. 若有未解析字，可從冒險故事輸出 `learned-characters.json`，手動補上 `zhuyin` 後由首頁重新匯入；也可按首頁的「升級舊版字庫」重新檢查。

完整故事字音會預先產生在 `ZhuyinAdventure/zhuyin-map.js`，首頁和 RPG 直接以 `<script>` 載入，因此以 `file://` 直接開啟 HTML 時也能升級，不依賴瀏覽器是否允許本機 `fetch()`。

## 使用方式

直接開啟根目錄的 `index.html`，再選擇要使用的學習功能或主選單工具。

所有內容都是純前端功能，不需要安裝套件或啟動後端。
