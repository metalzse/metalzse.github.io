# 注音互動故事引擎

這是一個給兒童閱讀的注音互動故事網頁。專案本身是一個簡單的故事引擎，可以透過替換 `story.js` 來體驗不同故事內容。

目前附上的 `story.js` 是一個範例故事，用來示範地點、門、道具、背包、鎖門條件與結局的寫法。

專案是純前端靜態網站，不需要安裝套件或執行建置流程，適合直接部署到 GitHub Pages。

## 如何開啟

直接用瀏覽器開啟 `index.html` 即可遊玩。

如果部署在 GitHub Pages，網站入口也是 `index.html`。更新線上檔案後，如果頁面看起來還是舊版，可以用 `Cmd + Shift + R` 強制重新整理。

## 撰寫自己的故事

第三方作者只需要替換 `story.js`，通常不需要修改 `index.html`、`engine.js` 或 `style.css`。

完整格式、範例與注意事項請看 [STORY_GUIDE.md](STORY_GUIDE.md)。

## 備用故事檔

除了目前載入的 `story.js`，資料夾中也保留了 `story_airport.js`、`story_dino.js`、`story_space.js` 等歷史故事檔。

這些檔案目前不會自動載入，保留它們是為了保存過去的故事版本，也方便之後加入「選擇故事」功能時可以重新使用。

## 專案功能

- 建立適合兒童閱讀的注音互動故事。
- 用同一套 `engine.js` 呈現不同故事。
- 支援分岔路線、返回上一個地點、道具收集、鎖門條件與結局。
- 支援點擊中文字，將字加入或移出已學會清單。
- 支援將學會的字清單輸出成文字檔。
- 支援選擇閱讀難度，控制注音顯示方式。
- 支援首次開啟或遊戲資訊更新時顯示規則說明。
- 使用瀏覽器保存玩家目前位置、背包、學會的字與閱讀難度。

## 閱讀難度

頁面提供三種閱讀難度：

- `簡單`：所有有注音的字都顯示注音。
- `練習`：未學會的字顯示注音，已學會的字隱藏注音。
- `挑戰`：全部隱藏注音。

難度選擇會存在瀏覽器的 `localStorage`，重新整理或下次開啟頁面時會保留。

## 學會的字

玩家可以點擊故事中的中文字，將該字加入學會清單。學會的字會顯示為較深的綠色。

再次點擊學會的字，會把該字從清單移除，文字也會回到預設黑色。

按下頁面上的 `輸出學會的字` 按鈕，可以下載 `learned-characters.txt`。檔案會列出目前學會的字與總字數。

## 遊戲資訊

首頁會在使用者第一次開啟時顯示遊戲資訊，說明遊戲規則、難度選擇，以及如何點選文字記錄學會的字。

遊戲資訊內容放在 `game-info.js`。當 `GAME_INFO.version` 變動時，使用者下次開啟頁面會再次看到遊戲資訊。

## 主要檔案

- `index.html`：網站入口，載入樣式、故事資料與互動引擎。
- `story.js`：故事內容，可替換成不同故事。
- `story_*.js`：歷史/備用故事檔，供日後故事選擇功能或手動替換使用。
- `game-info.js`：遊戲資訊內容與版本號。
- `engine.js`：互動引擎，負責畫面渲染、門的點選、背包、鎖門條件、學會字清單與進度保存。
- `style.css`：頁面樣式，包含大字體、注音、門、背包與提示區塊。
- `STORY_GUIDE.md`：第三方故事撰寫指南。
- `LICENSE`：MIT 授權條款。

## 技術說明

- 使用 HTML、CSS、JavaScript 製作，沒有外部框架。
- 使用 `<ruby>` 與 `<rt>` 顯示注音。
- 使用 `localStorage` 保存目前位置、背包道具、學會字清單、閱讀難度與已看過的遊戲資訊版本。
- 使用瀏覽器下載功能輸出學會的字清單，不需要後端服務。
- 使用網址 hash，例如 `#start`，表示目前故事地點。
- 故事資料以 `STORY` 物件集中管理，方便替換與擴充。
