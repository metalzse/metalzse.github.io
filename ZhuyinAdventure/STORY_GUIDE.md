# 故事撰寫指南

這份文件給想要替換 `story.js`、撰寫自己故事的作者使用。一般情況下，只需要改 `story.js`，不用修改 `engine.js`。

## 快速開始

1. 複製這個專案。
2. 用自己的內容替換 `story.js`。
3. 直接用瀏覽器開啟 `index.html` 測試。
4. 如果部署到 GitHub Pages 後看起來仍是舊版，請用 `Cmd + Shift + R` 強制重新整理。

如果需要修改首頁首次顯示的遊戲規則說明，請改 `game-info.js`，不要放在 `story.js`。

`story.js` 必須定義一個全域的 `STORY` 物件。因為 `story.js` 會先於 `engine.js` 載入，建議在檔案最上方保留這行：

```js
var t = window.t || function(text, zhuyin) { return { text: text, zhuyin: zhuyin }; };
```

## 遊戲資訊

`game-info.js` 用來管理首頁首次顯示的遊戲資訊，內容包含規則、難度選擇和學會字功能說明。

每次修改遊戲資訊內容時，請同步更新 `GAME_INFO.version`。使用者瀏覽器會記住看過的版本，只有第一次開啟或版本變動時才會再次顯示。

`game-info.js` 的文字也可以使用 `t("字", "注音")` 標記注音：

```js
const GAME_INFO = {
  version: "2026-07-03-1",
  title: [t("遊", "ㄧㄡˊ"), t("戲", "ㄒㄧˋ"), t("資", "ㄗ"), t("訊", "ㄒㄩㄣˋ")],
  sections: [
    {
      heading: [t("怎", "ㄗㄣˇ"), t("麼", "ㄇㄜ˙"), t("玩", "ㄨㄢˊ")],
      body: [
        [t("選", "ㄒㄩㄢˇ"), t("一", "ㄧˊ"), t("扇", "ㄕㄢˋ"), t("門", "ㄇㄣˊ"), t("繼", "ㄐㄧˋ"), t("續", "ㄒㄩˋ"), t("冒", "ㄇㄠˋ"), t("險", "ㄒㄧㄢˇ"), "。"]
      ]
    }
  ],
  closeLabel: [t("開", "ㄎㄞ"), t("始", "ㄕˇ")]
};
```

## 最小故事範例

可以先複製這份最小範例，再慢慢擴充房間、道具與路線。

```js
var t = window.t || function(text, zhuyin) { return { text: text, zhuyin: zhuyin }; };

const STORY = {
  id: "star-cookie-story",
  startRoom: "start",
  goal: [
    t("找", "ㄓㄠˇ"),
    t("到", "ㄉㄠˋ"),
    t("星", "ㄒㄧㄥ"),
    t("星", "ㄒㄧㄥ"),
    t("餅", "ㄅㄧㄥˇ"),
    t("乾", "ㄍㄢ"),
    "。"
  ],
  rooms: {
    start: {
      id: "start",
      title: ["🌟 ", t("星", "ㄒㄧㄥ"), t("星", "ㄒㄧㄥ"), t("廣", "ㄍㄨㄤˇ"), t("場", "ㄔㄤˇ")],
      description: [
        [t("你", "ㄋㄧˇ"), t("來", "ㄌㄞˊ"), t("到", "ㄉㄠˋ"), t("亮", "ㄌㄧㄤˋ"), t("亮", "ㄌㄧㄤˋ"), t("的", "ㄉㄜ˙"), t("廣", "ㄍㄨㄤˇ"), t("場", "ㄔㄤˇ"), "。"]
      ],
      doors: [
        {
          name: ["🚪 ", t("小", "ㄒㄧㄠˇ"), t("屋", "ㄨ")],
          description: [t("門", "ㄇㄣˊ"), t("後", "ㄏㄡˋ"), t("有", "ㄧㄡˇ"), t("甜", "ㄊㄧㄢˊ"), t("甜", "ㄊㄧㄢˊ"), t("的", "ㄉㄜ˙"), t("香", "ㄒㄧㄤ"), t("味", "ㄨㄟˋ"), "。"],
          to: "house"
        }
      ]
    },
    house: {
      id: "house",
      title: ["🏠 ", t("小", "ㄒㄧㄠˇ"), t("屋", "ㄨ")],
      description: [
        [t("桌", "ㄓㄨㄛ"), t("上", "ㄕㄤˋ"), t("有", "ㄧㄡˇ"), t("一", "ㄧˊ"), t("塊", "ㄎㄨㄞˋ"), t("餅", "ㄅㄧㄥˇ"), t("乾", "ㄍㄢ"), "。"]
      ],
      ending: ["⭐ ", t("找", "ㄓㄠˇ"), t("到", "ㄉㄠˋ"), t("了", "ㄌㄜ˙"), "！"],
      doors: [],
      back: {
        label: [t("回", "ㄏㄨㄟˊ"), t("到", "ㄉㄠˋ"), t("廣", "ㄍㄨㄤˇ"), t("場", "ㄔㄤˇ")],
        to: "start"
      }
    }
  }
};
```

## STORY 欄位

`STORY` 是整個故事的資料物件。

- `id`：故事代號，必填。請使用英文字母、數字、底線或連字號，例如 `space-story-1`。這個值會影響瀏覽器保存的進度與背包資料，換新故事時建議換新的 `id`。
- `startRoom`：起始房間 ID，必填。必須對應到 `rooms` 裡的一個房間。
- `goal`：冒險目標，建議填寫。會顯示在起始房間上方。
- `items`：道具清單，選填。故事沒有道具時可以省略。
- `rooms`：房間清單，必填。每個故事至少需要一個房間。
- `labels`：介面文字覆寫，選填。通常不需要使用。

## 文字與注音

故事中的文字通常是一個陣列。陣列裡可以放：

- `t("字", "注音")`：顯示文字並加上注音。
- 一般字串：適合放標點、空格、emoji，或不需要注音的文字。

```js
[t("你", "ㄋㄧˇ"), t("好", "ㄏㄠˇ"), "！"]
```

閱讀難度會自動控制注音顯示：

- `簡單`：全部顯示注音。
- `練習`：學會的字隱藏注音，未學會的字顯示注音。
- `挑戰`：全部隱藏注音。

作者不需要為閱讀難度另外寫資料。

## 房間欄位

每個房間放在 `STORY.rooms` 裡，key 通常和房間的 `id` 一樣。

```js
start: {
  id: "start",
  title: [],
  description: [],
  doors: []
}
```

- `id`：房間代號，必填。
- `title`：房間標題，必填。
- `description`：房間描述，必填。每一段是一個文字陣列。
- `doors`：可前往的門，必填。沒有門時可填空陣列 `[]`。
- `back`：返回用的門，選填。
- `givesItem`：進入房間時取得的道具 ID，選填。
- `ending`：結局文字，選填。

`description` 是「段落陣列」，所以格式會像這樣：

```js
description: [
  [t("第", "ㄉㄧˋ"), t("一", "ㄧ"), t("段", "ㄉㄨㄢˋ"), "。"],
  [t("第", "ㄉㄧˋ"), t("二", "ㄦˋ"), t("段", "ㄉㄨㄢˋ"), "。"]
]
```

## 門與返回

一般門會顯示在「可以走的門」下方。

```js
{
  name: [t("森", "ㄙㄣ"), t("林", "ㄌㄧㄣˊ")],
  description: [t("門", "ㄇㄣˊ"), t("後", "ㄏㄡˋ"), t("有", "ㄧㄡˇ"), t("樹", "ㄕㄨˋ"), "。"],
  to: "forest"
}
```

- `name`：門的名稱，必填。
- `description`：門的簡短描述，必填。
- `to`：要前往的房間 ID，必填。

返回門使用房間的 `back` 欄位：

```js
back: {
  label: [t("回", "ㄏㄨㄟˊ"), t("到", "ㄉㄠˋ"), t("廣", "ㄍㄨㄤˇ"), t("場", "ㄔㄤˇ")],
  to: "start"
}
```

## 道具與鎖門

如果故事需要道具，先在 `items` 裡定義。

```js
items: {
  golden_key: {
    name: ["🔑 ", t("金", "ㄐㄧㄣ"), t("鑰", "ㄧㄠˋ"), t("匙", "ㄕ˙")]
  }
}
```

房間可以用 `givesItem` 讓玩家進入時取得道具：

```js
key_room: {
  id: "key_room",
  title: [t("鑰", "ㄧㄠˋ"), t("匙", "ㄕ˙"), t("房", "ㄈㄤˊ")],
  givesItem: "golden_key",
  description: [
    [t("你", "ㄋㄧˇ"), t("找", "ㄓㄠˇ"), t("到", "ㄉㄠˋ"), t("鑰", "ㄧㄠˋ"), t("匙", "ㄕ˙"), "。"]
  ],
  doors: []
}
```

門可以用 `requires` 要求玩家先取得某個道具：

```js
{
  name: [t("金", "ㄐㄧㄣ"), t("色", "ㄙㄜˋ"), t("大", "ㄉㄚˋ"), t("門", "ㄇㄣˊ")],
  description: [t("鑰", "ㄧㄠˋ"), t("匙", "ㄕ˙"), t("發", "ㄈㄚ"), t("光", "ㄍㄨㄤ"), "，", t("門", "ㄇㄣˊ"), t("開", "ㄎㄞ"), t("了", "ㄌㄜ˙"), "。"],
  lockedDescription: [t("先", "ㄒㄧㄢ"), t("去", "ㄑㄩˋ"), t("找", "ㄓㄠˇ"), t("鑰", "ㄧㄠˋ"), t("匙", "ㄕ˙"), "。"],
  requires: "golden_key",
  to: "ending"
}
```

如果玩家還沒有 `requires` 指定的道具，這扇門會顯示為鎖住狀態。

## 內建介面文字

以下標準介面文字由 `engine.js` 提供，通常不需要放在 `story.js`：

- `冒險目標：`
- `可以走的門`
- `我的背包：`
- `重新開始`
- `背包還是空的。`
- `你得到了`
- `這扇門還不能開。`
- `輸出學會的字`

如果真的需要覆寫，可以加上 `labels`，只覆寫需要改的項目：

```js
labels: {
  doors: [t("你", "ㄋㄧˇ"), t("想", "ㄒㄧㄤˇ"), t("去", "ㄑㄩˋ"), t("哪", "ㄋㄚˇ"), t("裡", "ㄌㄧˇ"), "？"]
}
```

可覆寫的常用 key：

- `goal`
- `doors`
- `restart`
- `myBag`
- `emptyBag`
- `gotItem`
- `locked`
- `exportLearnedCharacters`

## 作者檢查清單

- `STORY.id` 是否是新的唯一 ID？
- `STORY.startRoom` 是否存在於 `rooms`？
- 每個門的 `to` 是否指向存在的房間？
- 每個 `requires` 是否對應到存在的道具 ID？
- 每個房間是否有 `id`、`title`、`description`、`doors`？
- 每個陣列、物件後面的逗號是否正確？
- 儲存後重新整理頁面，瀏覽器 Console 是否沒有錯誤？

## 給兒童故事的寫作建議

- 每段文字短一點，方便閱讀。
- 一個房間放 1 到 2 個選擇比較剛好。
- 門的描述可以提示孩子接下來會看到什麼。
- 鎖住的門要清楚提示孩子缺少什麼道具。
- 常用字可以重複出現，搭配學會字功能練習。
